# 19.2    Object-Oriented Design
オブジェクト指向でソフトウェアプロジェクトを設計するのは難しい、いや、非常に難しい。技術的な手法については、本書の他の箇所で詳しく解説している一方、設計手法については、他の書籍が本棚数メートルを埋めるに相当するほど存在するが、本書では、自分が設計に役立ててきたいくつかのヒントを紹介する。

抽象化を行い、クラスを設計し、カプセル化を構築する際には、常に「これはオブジェクト指向的な設計だろうか？」と自問自答する必要があります。この問いに100%「はい」と答えられることは稀です。しかし、現在の問題をきれいに解決するために必要な範囲でのみ抽象化を行うべきです。

自分が正しい方向に進んでいるかどうかを確認する手がかりを見つけることができます。SOLID原則（単一性、オープン性、リスコフ原則、インターフェース、依存性）に基づいて自分の設計を繰り返し検証したり、SRP、OCP、LSP、ISP、DIPといった略語を使ったりすることが役立ちます。

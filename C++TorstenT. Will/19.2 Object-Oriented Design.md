# 19.2    Object-Oriented Design
オブジェクト指向でソフトウェアプロジェクトを設計するのは難しい、いや、非常に難しい。技術的な手法については、本書の他の箇所で詳しく解説している一方、設計手法については、他の書籍が本棚数メートルを埋めるに相当するほど存在するが、本書では、自分が設計に役立ててきたいくつかのヒントを紹介する。

抽象化を行い、クラスを設計し、カプセル化を施しているときはいつも「今、私がやっているのはオブジェクト指向か？」と自問するだろう。100%「そうだ」と答えられることはまずない。それでも、抽象化は現在の問題をきれいに解決するための必要な範囲のみに留めるべきである。

自分が正しい方向に進んでいるかどうかを確認する手がかりがある。SOLID原則（単一性、オープン性、リスコフ原則、インターフェース、依存性）に基づいて自分の設計を繰り返し検証したり、SRP、OCP、LSP、ISP、DIPといった略語を使ったりすることが役立つ。
